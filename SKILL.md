# 四项穿梭车立体库 WCS 系统 / Four-Way Shuttle WCS System

## 项目概述 / Project Overview

- **客户 / Customer**: 宝洁黄埔工厂 / P&G Huangpu Plant
- **系统 / System**: WCS (Warehouse Control System) 控制四项穿梭车
- **上游 / Upstream**: Prime (WMS) 和 IAC (AGV 调度)
- **下游 / Downstream**: 四穿 PLC 控制系统
- **总库位 / Total Locations**: 1706个
- **储存层数 / Storage Levels**: 4层
- **进出口 / Ports**: 10个WH口 + 3个CFC口

## 系统架构 / System Architecture

```
┌─────────────┐      ┌─────────────┐      ┌─────────────┐
│   Prime     │      │    IAC     │      │    WES     │
│   (WMS)     │─────▶│  (AGV调度)  │─────▶│(仓库执行系统)│
└─────────────┘      └─────────────┘      └─────────────┘
                                                │
                                                ▼
                                        ┌─────────────┐
                                        │    WCS     │
                                        │(控制系统)   │
                                        └─────────────┘
                                                │
                                                ▼
                                        ┌─────────────┐
                                        │   四穿PLC   │
                                        │(设备控制)   │
                                        └─────────────┘
```

### 系统组件 / Components

- **Prime**: WMS仓储管理系统，负责库存管理和任务下单
- **IAC**: AGV调度系统，负责AGV任务调度
- **WES**: Warehouse Execution System，仓库执行系统
- **WCS**: Warehouse Control System，仓库控制系统
- **四穿PLC**: 四项穿梭车可编程逻辑控制器

## 核心概念 / Core Concepts

### 名词解释 / Terminology

| 术语 / Term | 说明 / Description |
|-------------|---------------------|
| Prime排位 | 上游Prime记录库存的单位，一个排位对应一个托盘位 |
| 地台板条码 | 对应WES的容器编号 (Load Number)，贴在托盘四个面 |
| 四穿库 | 四项穿梭车立体库（双向穿梭车），用于自动化存取 |
| Induction | 入库任务单，从Prime下发到WCS的入库指令 |
| Pick_Request | 出库任务单，从Prime下发到WCS的出库指令 |
| ULID | 容器ID，用于唯一标识每个托盘容器 |
| BCP | 暂存排位，用于临时存放无法上架的物料 |
| 4WS_DUMMY | 需要进四穿库的标记，区分普通地面库和四穿库 |
| WES | Warehouse Execution System，仓库执行系统 |
| AGV | Automated Guided Vehicle，自动导引车 |

### 货位结构 / Location Structure

- **深位组 / Depth Group**: 6深位库位组（每组可存6托）
- **层 / Level**: 优先推荐层配置，系统自动计算最优层
- **库位编码 / Location Code**: 格式 ASRS-区域-排-层-位

#### 库位编码规则

```
ASRS-[区域代码]-[排号]-[层]-[位]
例：ASRS-A-01-01-01 表示A区1排1层1位
```

### 批次属性 / Batch Attributes

| 属性 | 说明 | 示例 |
|------|------|------|
| LOTNUM | 批号 | LOT20240401 |
| FTPCOD | 托盘规格 | FP001 |
| MANDTE | 生产日期 | 2024-04-01 |
| INVSTS | 库存状态 | U(正常)/Q(待验) |
| EXPIRE_DTE | 过期日期 | 2025-04-01 |

## 核心功能 / Core Functions

### 1. 库存管理 / Inventory Management

#### 功能说明

- 货位管理（货架、货位状态）
- 容器管理（容器类型、容器库存）
- 物料类型管理（item family设置）
- 批次属性管理
- 商品信息管理（从Prime同步）

#### 库存状态 / Inventory Status

| 状态 | 说明 | 可出库 |
|------|------|--------|
| U | 正常库存 | ✅ |
| Q | 待检库存 | ❌ |
| H | Hold库存 | ❌ |
| X | 冻结库存 | ❌ |

### 2. 任务管理 / Task Management

#### 任务类型

| 类型 | 说明 | 生命周期 |
|------|------|----------|
| Induction | 入库任务 | 新建→受理→执行→完成/失败 |
| Pick_Request | 出库任务 | 新建→受理→执行→完成/失败 |
| Move | 库内倒库任务 | 新建→受理→执行→完成/失败 |

#### 任务状态

| 状态 / Status | 可取消 / Cancel | 可改优先级 / Priority | 可强制 / Force |
|---------------|----------------|----------------------|----------------|
| 新建 / New | ✅ | ✅ | ✅ |
| 受理 / Assigned | ❌ | ❌ | ✅ |
| 执行 / Executing | ❌ | ❌ | ✅ |
| 完成 / Completed | ❌ | ❌ | ❌ |
| 失败 / Failed | ❌ | ❌ | ❌ |

#### 优先级规则

- 范围：1-50，数值越低优先级越高
- 自动升级：每10分钟降2级
- 紧急任务可手动提升优先级

### 3. 策略配置 / Strategy Configuration

#### 上架策略

1. 同SKU同批次 → 优先放同一库位组
2. 空位组最深层 → 优先使用最 深位置
3. 异常物料 → 自动转 BCP 暂存排位

#### 分配策略

- 按托盘号分配（精确匹配）
- 按Batch + Footprint分配（按日期+规格）
- 按Footprint分配（按规格，最早批次）

#### 倒库策略

- 外深位托盘先出（外层先出）
- 同深度按入库时间先后
- 空闲时自动触发理货

## 入库流程 / Inbound Flow

### 操作步骤 / Step-by-Step

```
1. 材料收货后上架，按F6键
2. 选择"1 Directed定向工作"
3. 系统自动推荐排位
4. 检查4WS_DUMMY=需进四穿库
5. 叉车到入库口（指示灯绿色）
6. RDT扫描入货口排位
7. 放输送带完成入库
```

### 详细流程图

```
Prime下单 → IAC接收 → RCS搬运到缓存位 → WES分配入库口 → 
PLC检测 → WCS识别 → WES判断成功/失败 → 回传Prime
```

### 入口分配规则

| 入口 | 适用情况 |
|------|----------|
| 1-8 | AGV工作站入库 |
| 9 | 叉车01入库/AGV异常 |
| 10 | 叉车02入库/AGV异常 |

### 异常处理 / Exception Handling

| 异常代码 | 描述 | 处理方案 |
|----------|------|----------|
| E101 | BCR扫码异常 | 输送线回退→异常口 |
| E102 | 超高超宽 | 输送线回退→异常口 |
| E103 | 托盘倾斜 | 人工整理后重试 |
| E104 | 物料超重 | 转地面排位 |
| E105 | 物料超高 | 转BCP排位 |
| E106 | 类型不允许 | 转BCP排位 |
| E107 | 无库位 | 转地面排位 |
| E108 | 输送带卡料 | 复位/手动 |
| E109 | 重复扫码 | 人工确认跳过 |
| E110 | 卡托盘 | 复位/强制失败/手动 |

## 出库流程 / Outbound Flow

### 操作步骤 / Step-by-Step

```
1. Prime下Pick_Request出库任务
2. WES根据指令分配库存（按托盘号/batch/footprint）
3. WCS调度四穿车下架
4. 输送带到出库口
5. 叉车/AGV取货
6. 出库完成确认
```

### 详细流程图

```
Prime下单 → WES分配 → WCS调度 → 四穿取货 → 
输送带运输 → 出库口 → 人工/AGV取货 → 
pick_confirm回传 → Prime确认
```

### 出口分配规则

| 出口 | 适用情况 |
|------|----------|
| 1-8 | AGV工作站出库/AGV异常 |
| 9 | 叉车01出库/叉车异常 |
| 10 | 叉车02出库/叉车异常 |

### 异常处理 / Exception Handling

| 异常代码 | 描述 | 处理方案 |
|----------|------|----------|
| E201 | 质量状态变更 | 继续执行→move_error |
| E202 | 取消失败 | 继续执行→move_error |
| E203 | 无匹配库存 | 检查Prime库存分布 |
| E204 | 出库超时 | 强制完成/取消 |
| E205 | 托盘损坏 | 检查托盘状态 |

## WES系统维护 / WES Maintenance

### Item Family 设置

1. 登录WES系统
2. 进入"商品类型"菜单
3. 选择需要设置的item family
4. 编辑属性：
   - 是否四穿库储存：是/否
   - 优先推荐层：1-4层

### 优先层设置

1. 商品类型 → 选择物料
2. 编辑 → 优先推荐层
3. 选择层数（1-4）
4. 保存设置

### 任务超时处理

1. 登录WES系统
2. 查询滞留任务（滞留时间>30min）
3. 分析原因：
   - 设备故障
   - 物料异常
   - 人工阻塞
4. 执行操作：
   - 取消任务
   - 强制完成
   - 强制失败

## FAQ / 常见问题

**Q1: 4WS_DUMMY是什么？**
A: 4WS_DUMMY是材料是否需要进入四穿库的标记。当物料需要存入四穿立体库时，系统自动标记4WS_DUMMY，并推荐入库口9或10。

**Q2: 入库口指示灯红色是什么状态？**
A: 指示灯红色表示当前出库状态，不允许放货。需要等待指示灯变绿后方可放货入库。

**Q3: 任务滞留超过30分钟怎么办？**
A: 登录WES系统，检查任务滞留原因，根据情况执行取消、强制完成或强制失败操作。

**Q4: 如何查询库存？**
A: 登录WES系统，选择"库存"菜单，输入Load Number（托盘号）即可查询库存详细信息。

**Q5: 如何处理BCR扫码失败？**
A: 检查条码是否清晰、镜头是否脏污，必要时使用手动输入或更换条码。

**Q6: 什么是 BCP 排位？**
A: BCP是暂存排位，用于临时存放无法正常上架的物料，如物料类型不允许、无库位等情况。

**Q7: 如何切换出入口模式？**
A: 现场旋转切换按钮，PLC→WCS→WES→IAC逐层传递，锁闭当前工作站，任务完成后切换生效。

**Q8: 托盘号在哪里查看？**
A: 托盘号（ULID）贴在托盘四个面，条码格式为4位数字+字母组合。
