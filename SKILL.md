# 四项穿梭车立体库 WCS 系统 / Four-Way Shuttle WCS System

## 项目概述 / Project Overview

### 基本信息 / Basic Info

| 项目 | 内容 |
|------|------|
| 客户 | 宝洁黄埔工厂 / P&G Huangpu Plant |
| 系统 | WCS (Warehouse Control System) |
| 上游 | Prime (WMS) + IAC (AGV调度) |
| 下游 | 四穿 PLC 控制系统 |
| 总库位 | 1706个 |
| 储存层数 | 4层 |
| 进出口 | 10个WH口 + 3个CFC口 |

### 系统架构 / Architecture

```
Prime (WMS) → IAC (AGV) → WES → WCS → 四穿PLC → 设备
```

## 核心概念 / Core Concepts

### 名词 / Terminology

| 术语 | 说明 |
|------|------|
| Prime排位 | 上游Prime记录库存的单位 |
| 地台板条码 | 对应WES的容器编号 (Load Number) |
| ULID | 容器ID |
| BCP | 暂存排位 |
| Induction | 入库任务单 |
| Pick_Request | 出库任务单 |
| 4WS_DUMMY | 需要进四穿库的标记 |

### 货位结构 / Location

- 深位组：6深位库位组（每组可存6托）
- 库位编码：ASRS-区域-排-层-位

---

## 核心功能 / Core Functions

### 1. 库存管理

- 货位管理
- 容器管理
- 物料类型管理
- 批次属性（LOTNUM, FTPCOD, MANDTE, INVSTS, EXPIRE_DTE）

### 2. 任务管理

- 入库：Prime→Induction→WES→WCS→四穿
- 出库：Prime→Pick_Request→WES→WCS→四穿
- 任务状态：新建→受理→执行→完成/失败
- 优先级：1-50，数值越低越高

### 3. 策略配置

- 上架：同SKU同批次→空位组最深层
- 分配：按Prime要求+WES最优
- 倒库：外深位托盘先出
- 优先级升级：每10分钟降2级

## 入库流程 / Inbound Flow

### 操作步骤

1. 材料收货后上架，按F6键
2. 选择"1 Directed定向工作"
3. 系统自动推荐排位
4. 4WS_DUMMY=需进四穿库
5. 叉车到入库口（指示灯绿色）
6. RDT扫描入货口排位
7. 放输送带完成

### 异常处理

| 异常 | 方案 |
|------|------|
| BCR扫码异常 | 输送线回退→异常口 |
| 超高超宽 | 输送线回退→异常口 |
| 物料类型不允许 | 转BCP排位 |
| 无库位 | 转地面排位 |
| 卡托盘 | 复位/强制失败/手动 |

---

## 出库流程 / Outbound Flow

### 操作步骤

1. Prime下Pick_Request
2. WES分配库存（按托盘号/batch/footprint）
3. WCS调度下架
4. 输送带到出库口
5. 叉车取货

### 异常处理

| 异常 | 方案 |
|------|------|
| 质量状态变更 | 继续执行→move_error |
| 取消失败 | 继续执行→move_error |
| 无匹配库存 | 检查Prime库存 |

## WES系统维护 / Maintenance

### 设置Item Family

1. WES系统 → 商品类型
2. 选择item family
3. 编辑 → 是否四穿库储存（是/否）

### 设置优先层

1. 商品类型 → 编辑
2. 优先推荐层选择层数

### 任务超时处理

1. 登录WES，滞留时间>30min
2. 查询原因
3. 取消/强制完成

## FAQ

**Q: 4WS_DUMMY是什么？**
A: 材料需进四穿库，系统推荐入库口9/10

**Q: 入口指示灯红色？**
A: 出库状态，不允许放货

**Q: 任务>30分钟？**
A: 登录WES检查，取消或强制完成

**Q: 如何查库存？**
A: WES系统-库存-输入Load Number
