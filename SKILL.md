# 四项穿梭车立体库 WCS 系统

## 项目概述

- **客户**：宝洁黄埔工厂
- **系统**：WCS (Warehouse Control System) 控制四项穿梭车
- **上游**：Prime (WMS) 和 IAC (AGV 调度)
- **下游**：四穿 PLC 控制系统
- **总库位**：1706个
- **储存层数**：4层
- **进出口**：10个WH口 + 3个CFC口

## 核心概念

### 名词解释

| 术语 | 说明 |
|------|------|
| Prime 排位 | 上游 Prime 记录库存的单位（库区/排/库位组/货位） |
| 地台板条码 | 对应 WES 的容器编号 (Load Number) |
| 四穿库 | 四项穿梭车立体库（双向穿梭车） |
| Induction | 入库任务单 |
| Pick_Request | 出库任务单 |
| ULID | 容器ID |

### 货位结构

- **深位组**：6深位库位组（每组可存6托）
- **层**：优先推荐层配置
- **库位编码**：格式 ASRS-区域-排-层-位

## 核心功能

### 1. 库存管理

- 货位管理（货架、货位）
- 容器管理（容器类型、容器库存）
- 物料类型管理
- 批次属性（LOTNUM, FTPCOD, MANDTE, INVSTS, EXPIRE_DTE）

### 2. 任务管理

- 入库任务：Prime 下发 Induction → WES 生成上架任务 → WCS 调度四穿
- 出库任务：Prime 下发 Pick_Request → WES 分配库存 → WCS 调度下架

### 3. 策略配置

- **上架策略**：同SKU同批次优先 → 空位组最深层 → 异常
- **分配策略**：按 Prime 要求 + WES 最优
- **倒库策略**：外深位托盘先出

## 系统架构

### 层级结构

```
Prime (WMS) → IAC (AGV调度) → WES (仓储执行) → WCS (设备控制) → 四穿PLC → 设备
```

## 设备故障排查

### 常见故障

| 故障 | 可能原因 | 解决方法 |
|------|----------|----------|
| 四穿不运行 | 电源/控制柜 | 检查急停/电源 |
| 输送带不转 | 电机/变频器 | 检查驱动/电机 |
| BCR不响应 | 条码损坏/镜头脏 | 清洁镜头/换条码 |
| PLC报警 | 传感器异常 | 检查传感器 |
| 任务卡住 | 通信中断 | 重启通信 |

### 故障处理流程

1. 观察现场报警灯/大屏信息
2. 记录故障代码
3. 查看WCS日志定位
4. 检查对应传感器/设备
5. 必要时复位重启
6. 记录故障和处理过程

## 异常处理方案

### 入库异常

1. BCR 扫码通过但未推荐到库位 → 库存记上架暂存区 → 异常回退任务
2. 异常回退后，WES 向 IAC 下发去异常排位的 induction
3. 异常处理完成后，自动下发入四穿库的 induction

### 出库异常

1. 质量状态变更（U → 非U）→ 标记异常 → 回传 move_error
2. 取消失败 → 继续执行 → 回传 move_error
3. 异常作站台显示托盘号和原因

### 任务处理

- 新建状态：可取消、可修改优先级
- 受理/执行状态：部分可取消
- 优先级：数值越低优先级越高

### 现场异常处理（OPL）

| 异常问题 | 操作指引 |
|----------|----------|
| 外检异常 | 严重：拉回车间整理；不严重：叉车司机简单整理后重新扫描入库 |
| 无上架任务 | 确认排位是否ASRS进口排位，重新扫描入库 |
| 无条码信息 | 条码破损，重新换条码 |
| 系统显示 ASRS_DUMMY | 取消供线任务，重新扫描入库 |
| 商品分类不允许入四穿 | 材料转回正常排位储存 |

### 出入口操作要求

| 指示灯 | 状态 | 操作 |
|--------|------|------|
| 红色 | 出库状态 | 不允许放货 |
| 绿色 | 入库状态 | 可以放货 |
| 红绿闪烁 | 取货提醒 | 有货送出，可取走 |

### 收货检查流程

**场景1：整板送货+供应商标签**
1. 卸货划掉供应商ULID
2. 检查超板/缠膜 → 超板需整理，拒收
3. 检查地台板破损 → 拒收贴"out of usu"
4. 检查条码完整性 → 缺则贴4联标签
5. 收货使用ULID
6. 上库前检查底部 → 破损需换板
7. 上架用"定向" → 4WS_DUMMY送立库9/10口

**场景2：整板送货+仓库打印标签**
- DA打印标签 → 划掉ULID → 其他同场景1

**场景3：散装送货**
- 使用合格地台板 → 其他同场景1

**检查要点**
| 检查项 | 要求 |
|----------|------|
| 地台板 | 4面有条码，破损拒收 |
| 超板 | 整理后仍超板拒收 |
| 缠膜 | 单层，尾巴收里面 |
| 标签 | 不翘起、不外露、无划痕 |

### 标签条码位置

**ULID贴标位置**
- 位置：地台板4个角落各贴1张
- 要求：4面ULID信息完全一致

**生产线操作**
| 操作 | 标签要求 |
|----------|----------|
| 供线 | 使用地台板ULID，保留收货标签在实物上 |
| 退料 | 扫描地台板ULID，退料标签+原收货标签贴在实物上 |

### 退料接收标准

**退料要求（生产线）**
| 要求 | 说明 |
|----------|------|
| 地台板外形 | 不超板，符合四穿库要求 |
| 标签信息 | 与实物一致（材料代码、批号、数量） |
| 地台板 | 用原供应商板，没有就用仓库板 |
| 数量 | 不能大于PRIME叠堆数量 |
| 上架 | 用"定向"，4WS_DUMMY送立库9/10口 |

**退料要求（供线叉车司机）**
| 要求 | 说明 |
|----------|------|
| 检查外形 | 确保不超板 |
| 检查标签 | 与实物一致 |
| 地台板 | 用原供应商板/仓库板 |

**检查要点**
| 检查项 | 要求 |
|----------|------|
| 标签 | 不翘起、不外露、无划痕 |
| 材料/缠膜 | 不能超板 |
| 周转材料 | 只能用WH塑料地台板 |
| 缠膜 | 212方式，顶部底部收好尾巴 |

### 退料操作

| 操作 | 说明 |
|----------|------|
| 退料任务 | 叉车司机接到退料任务后，按F6 |
| 4WS_DUMMY | 产品叉运到四穿库入库口（指示灯为绿色） |
| 入库 | 用RDT扫描入货口排位，放在输送带上完成进库 |

### 指定容器出库

**操作流程**
| 步骤 | 操作 |
|----------|------|
| 1 | WES系统打开指定容器出库，输入Load Number |
| 2 | 按查询，选择出库口 |
| 3 | 叉车异常/出库工作站01=9#口，02=10#口 |
| 4 | 按保存（出库口需手动改为出库状态） |

**生产线供线**
- 选择AGV出库/异常工作站07（不用改出库状态）
- 任务列表显示任务状态
- 货物送出口，叉车司机使用"整板库存转移"，扫描产品地台板条码，把货物库存转移到仓库排位

**核对**
- 当班需要检查Load Number在PRIME和WES库存是否一致

### 材料入库操作

**入库流程**
| 步骤 | 操作 |
|----------|------|
| 1 | 材料收货后上架，按F6键 |
| 2 | 选择"1 Directed（定向工作）"，系统自动推荐排位 |
| 3 | 4WS_DUMMY = 材料需要在四穿库储存 |
| 4 | 叉车司机把材料叉运到四向库对应入货口 |
| 5 | 入货口指示灯为绿色可以入库 |
| 6 | 用RDT扫描入货口排位，放在输送带上完成进库 |

**入库口对应**
| 入口 | 对应业务 |
|----------|------|
| PM - 9# 10# | 仓库材料收货/提货 |
| WIP - 1#~8# | 主线材料供线/退料/CFC2 WIP |

### 材料出库操作（指定条码）

**指定板出库**
| 步骤 | 操作 |
|----------|------|
| 1 | Prime - Pallet Request Operations 输入需要出库的Load Number，按Find |
| 2 | 点击Process |
| 3 | 输入需要下架的排位，按OK |
| 4 | 仓库9/10下架后排位ASRSRPM_MR（下架前把9或10改为出库） |
| 5 | 输入生产线号就会推到对应的出货口 |
| 6 | 显示Success完成 |
| 7 | WES系统任务列表就会显示下架信息 |

### WES任务>30min处理

**处理步骤**
| 步骤 | 操作 |
|----------|------|
| 1 | 登录WES任务界面 |
| 2 | 在滞留时间选择">30min"，点击查询 |
| 3 | 调查Prime库存和操作记录，核对实物真实库存位置 |

**任务状态处理**
| 任务状态 | 处理方式 |
|----------|----------|
| 新建 | 不需要执行则WES取消任务 |
| 执行 | 不需要执行则WES强制完成任务 |
| 执行中（非新建） | 检查4穿库设备是否有问题 |

**其他操作**
- 任务删除：任务状态=新建时WES可取消，其他状态不能取消
- 强制完成任务：出库任务强制完成后，WES库存会清空

### 软件异常处理流程（详细）

**入库异常**
| 异常 | 处理 |
|----------|----------|
| 外形检测异常 | 线下整理后重新扫描，或转BCP排位 |
| BCR扫码任务不存在 | 重新扫码下发，或转BCP排位 |
| BCR扫不到码 | 调换一面继续，或转BCP排位 |
| 物料类型不允许入四穿 | 转BCP排位，检查item family |
| 没有可用库位 | 转入库旁地面排位 |
| 相同托盘号库存 | 转地面排位，调查差异 |
| 商品数量大于板规 | 转地面排位，维护footprint |
| 入库时卡托盘 | 复位/强制失败/手动完成 |

**出库异常**
| 异常 | 处理 |
|----------|----------|
| 没有匹配库存 | 检查Prime库存比对 |
| 质量状态变更 | 继续执行，回传move_error |
| 取消不允许 | 继续执行，回传move_error |
| 出库时异常 | 强制完成/手动处理 |

## 接口对接

### 与 Prime 接口

- 入库：Induction (ULID, PRTNUM, PRTQTY)
- 出库：Pick_Request (托盘号/批次/ footprint code)
- 回传：pick_confirm, move_error

### 与 IAC/AGV 接口

- 入库：IAC WCS → 快仓 RCS → 缓存位 → 四穿入库口
- 出库切换：现场旋转按钮 → PLC → WCS → WES → IAC

### 优先级升级

- 范围：1-50
- 开始优先级：50，结束优先级：1
- 步长：每10分钟降2

## 闲时理货

### 逻辑

1. 按库位组维度保证巷道均衡
2. 不在推荐层的托盘→推荐层
3. 单深位有货→同批次多深位
4. 多深位未满→同SKU同批次空闲位

### 操作

- 创建理货单 → 校验库存 → 生成出库单
- 批量执行/取消理货单

## 日常检查项

### 叉车司机检查
- 检查地台板4面条码完整性
- 检查材料/缠膜不超板
- 检查标签不翘起、无划痕
- 上架前检查地台板底部

### 仓库DA检查
- 核对实物与标签信息一致
- 检查地台板是否破损
- 检查缠膜是否符合212标准

- 检查任务滞留时间>30min
- 核对Prime与WES库存一致性
- 检查设备异常报警
- 任务强制完成后核对库存

## 常见问题FAQ

**Q: 4WS_DUMMY是什么意思？**
A: 代表材料需要进四穿库储存，系统会推荐入库口9/10

**Q: 任务为什么被取消？**
A: 新建状态可取消，执行中需强制完成或检查设备

**Q: 入库口指示灯红色可以放货吗？**
A: 不可以，红色代表出库状态

**Q: 如何查询库存位置？**
A: 在WES系统-库存界面输入Load Number查询

**Q: 任务>30min未完成怎么办？**
A: 登录WES查询滞留时间，检查原因后取消或强制完成
